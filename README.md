# Проект «Key-Learning»
Данный репозиторий - результат упорного качество-оринетированного труда [сплоченной](https://vk.com/wall-152990521_2655) команды 
высококвалифицированных кросс-функциональных прогрессивных и продуктивных разработчиков. 

Проект напревлен на создание веб-приложения для обучения и отработки взаимодействий с компьютером (клавиатурой). 
Цель – в игровом формате показать возможности клавиатуры и связанные с ней техники; обучить человека быстрее и проще взаимодействовать с компьютером.

## Правила по работе с репозиторием
1. При отправке кода в feature-branch общего репозитория происходит кросс-валидация
2. Проверяющий для pull-request назначается менеджером в YouGile (чекер)
3. В репозитории ветка release для существующей версии (результаты спринтов), на ней CI\CD.
4. Ветка main для хранения рабочей версии
5. Для фичи создаётся ветка feature-branch, в неё коммиттят все ответственные. Затем она вливается в main
6. После вливания feature-branch в main создаётся t_feature-branch для теста.
7. Комментарий к коммиту обязателен

### Применение миграций
Для вывода справки необходимо выполнить следующую команду:
```bash
./gradlew FlywayMigrator
```
Для применения миграций введите:
```bash
./gradlew FlywayMigrator --args=migrate
```
Для вывода текущей миграции в бд и количества миграций, которые ожидают применения, введите:
```bash
./gradlew FlywayMigrator --args=info
```
Для синхронизации миграций (выравнивания чек-сумм) введите:
```bash
./gradlew FlywayMigrator --args=repair
```
Для полной очистки базы данных введите:
```bash
./gradlew FlywayMigrator --args=clean
```

### Запуск приложения
Выполните задачу
```bash
./gradlew run
```

### Выполнение тестов
```bash
./gradlew test --rerun-tasks
```

### Выполнение проверок качества исходного кода
```bash
./gradlew detekt
./gradlew ktlintCheck
```
